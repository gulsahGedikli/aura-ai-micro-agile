# AURA-AI-MICRO-AGILE 
> **A Pioneer Software Engineering Methodology for the AI Era**

---

## Overview
**AURA-Aİ-MICRO-AGILE** is an ultra-lightweight, "back-to-basics" framework designed to synchronize human architectural oversight with the exponential generation speed of AI. 

In an era where AI can produce code in seconds, traditional 2-week sprints have become a bottleneck. AURA replaces heavy ceremonies with **4-24 hour Pulses**, ensuring that velocity never compromises structural integrity.

## The Core Disciplines
1. **The Pulse (4-24h):** Short, rhythmic development cycles aligned with AI output capacity.
2. **Context Injection:** The practice of feeding "crystallized" architectural constraints to the AI before any code is generated.
3. **The Sift:** A required human-led review to remove unnecessary complexity from AI-generated code and redundant logic.
4. **Validation-First:** Success criteria and test cases must be finalized and "locked" before the Agent begins execution.

## Roles
| Role | Responsibility |
| :--- | :--- |
| **Architect (Human)** | System Design, Prompt Contracts, Strategic Vision |
| **Agent (AI)** | Rapid Code Generation, Unit Testing, Documentation |
| **Validator (Hybrid)** | Quality Gate, Security Check, and Logic Approval |

## Repository Structure
- `/templates`: Standardized files for AI-Human communication.
- `/examples`: Real-world scenarios
- `/workflow`: Automation scripts for Pulse cycles.
  
## Quick Start
1. **Prepare:** Copy the `templates/PULSE_TEMPLATE.md` file.
2. **Inject:** Define your **Objective** and **Validation Criteria** for the AI Agent.
3. **Generate:** Run the Pulse using your preferred AI tool (Cursor, Claude, Copilot, etc.).
4. **Sift:** Review the output, prune the unnecessary, and merge into the main branch.
